# Scenario Specs

Planned scenario set:
- `low`: low demand baseline.
- `medium`: medium demand baseline.
- `high`: high demand baseline.
- `demo`: default hackathon demo, mapped to the real-city OSM scenario.
- `city`: explicit real-city scenario alias.

Each scenario currently defines:
- simulation time window in `.sumocfg`,
- route file inputs,
- GUI behavior and runtime defaults in `sumo_contract.json`.

Real-city GUI defaults:
- GUI settings file: `sumo/scenarios/city_3d.settings.xml`
- OSG renderer: disabled by default for `demo` and `city` in `sumo_contract.json`

Generated runtime artifacts:
- `*_rsu_pois.add.xml` files are generated by `sumo/run_sumo_pipeline.py` during GUI runs.
- These files are not source-of-truth assets and are ignored via `.gitignore`.
